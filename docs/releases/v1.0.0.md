# PCA9555 v1.0.0

Initial stable production release of the PCA9555 16-bit I/O expander library for ESP32-S2 / ESP32-S3 using the Arduino framework under PlatformIO.

## Highlights

- Production-focused driver architecture with deterministic behavior, explicit `Status` returns, and no `Wire` ownership inside library code.
- Managed synchronous health tracking with `UNINIT`, `READY`, `DEGRADED`, and `OFFLINE` driver states.
- Full 16-bit PCA9555 control surface: inputs, outputs, direction, polarity inversion, single-pin helpers, and direct register access.
- Cached bit-manipulation helpers for output, direction, and polarity masks in single paired-register I2C bursts.
- Recoverable runtime state: `recover()` re-applies the current live configuration instead of reverting to the original `begin()` inputs.
- Rich bring-up CLI example with hardware-oriented inspection, direct register diagnostics, safe self-test, stress testing, and exact output-pattern drive.

## Added

- Injected transport callbacks via `Config::i2cWrite` and `Config::i2cWriteRead`.
- Device presence verification in `begin()` using the configuration register pair.
- Configurable POR-default enforcement through `Config::requireConfigPortDefaults`.
- Symmetric readback helpers:
  - `readOutput()`
  - `readOutputPin()`
  - `getPortConfiguration()`
  - `getPortPolarity()`
  - `getPinDirection()`
  - `getPinPolarity()`
- Runtime inspection via `SettingsSnapshot` and `getSettings()`.
- Pair-bounded bulk register helpers: `readRegisters()` and `writeRegisters()`.
- Bit manipulation API:
  - `setOutputBits()`
  - `clearOutputBits()`
  - `toggleOutputBits()`
  - `togglePin()`
  - `configureInputBits()`
  - `configureOutputBits()`
  - `setInvertBits()`
  - `clearInvertBits()`
- Example common helpers for CLI I/O, health display, bus diagnostics, and transport adaptation.
- Repository checks for CLI contract and timing guard behavior.

## Example CLI Improvements

- Added descriptive aliases alongside short commands for bring-up use.
- Added register-pair commands for low-level diagnostics.
- Added single-pin inspection and full 16-pin summary views.
- Added `pattern <M>` to drive an exact 16-bit output image while forcing all pins to output mode.
- Expanded help text with explicit definitions for:
  - `port`
  - `pin`
  - `polarity`
  - 16-bit mask `M`
- Expanded `selftest` and `stress_mix` so they exercise output, direction, polarity, and mask-based helper flows.

## Behavioral Fixes And Hardening

- `recover()` now restores the latest runtime output, direction, and polarity state.
- Direct register access now keeps cached shadow state synchronized.
- Invalid `Port` values are rejected explicitly.
- Overlong CLI input is discarded instead of being executed in truncated form.
- Example parser helpers reject malformed numeric input instead of accepting partial tokens.

## Validation

- `pio test -e native`
- `pio run -e esp32s3dev`
- `pio run -e esp32s2dev`
- `python tools/check_cli_contract.py`
- `python tools/check_core_timing_guard.py`

## Notes

- `examples/common/` remains example-only glue and is not part of the library API.
- `library.json` is the version source of truth for this release.
- `include/PCA9555/Version.h` remains auto-generated and should not be edited manually.
